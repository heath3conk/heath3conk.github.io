JavaScript Objects Cheatsheet


        1   var name = "Heather";
Objects can be simple variables. 

         1  var meals = {
         2    breakfast: {
         3      baked_goods: ["toast", "muffin", "coffee cake"],
         4      cereal: ["granola", "Capn Crunch"],
         5      griddle: ["pancakes", "french toast", "hash browns"],
         6      eggs: ["scrambled", "over easy", "quiche"]
         7    },
         8    lunch: {
         9      sandwiches: ["PB&J", "roast beef", "tuna salad"],
        10      soup: ["tomato", "chicken curry", "chili", "beef stew"],
        11      salad: ["chopped", "greens only", "everything in the garden"]
        12    }
        13  };

Objects can also hold other objects and/or functions. The properties inside the object are separated by commas. The property names can be formatted like symbols in Ruby, without quotes and followed by a colon, as is breakfast: on line 2 and soup on line 10.

The properties and their values function like key/value pairs in a Ruby hash. 


        14  meals.dinner = {};
        15  meals.dinner.entrees = ["roast chicken", "meatloaf", "pork chops", "veggie lasagne"];
        16
        17  console.log(meals.dinner);

         => { entrees: [ 'roast chicken', 'meatloaf', 'pork chops', 'veggie lasagne' ] }
The meals object contains properties that are themselves objects. To add another one first requires that you show it is an object and give it a name (line 1). Then you can provide the new entrees properties its own value (line 2). 

        18  console.log(meals["breakfast"]["baked_goods"][1]);
        19  console.log(meals.breakfast.baked_goods[1]);
        20  console.log(meals[breakfast][baked_goods][1]);

         =>  muffin
         =>  muffin
         =>  ReferenceError: breakfast is not defined

The first two lines are valid ways to access entries in a object. Either the bracket notation on line or the dot notation on line 2 work fine. Both print the value muffin to the screen.

The third line raises an error that breakfast is not defined. (It would raise the same error for baked_goods, if it got that far.)
